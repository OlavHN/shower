Languages, frameworks & UX

Noone controls the web. When someone (like Mozilla) has an idea, they implement it as a prototype and propose it for standardization If the prototype becomes popular, the browser vendors and others collaborate on a standard that everyone can implement webRTC is a complex beast that makes it possible for browsers to communicate directly. Google prototyped and proposed it. Others are now implementing it. If someone doesn't like the proposal, like how Microsoft is afraid of webGL, they can either unwillingly implement it or have a browser that is worse than the rest. This is innovation on the web. Everyone innovating and everyone collaborating. That is what the webAPIs are all about. Moving the web to mobile.
Of course, while this sounds great and idealistic in theory, how is this actually going to happen?

Turtles all the way

A webapp built on shims built on webviews built in native app wrappers built on operating systems ..

There is something called the Ubuntu Phone. A brand new phone that builds on ubuntu and competes with high end Andoird and iPhone. Sexy stuff, but iPhone sexy? Mozilla realizes Apple has a few years on them, so initialy they target the two billion people that is using fragmented, broken andoirds, feature phones or no phone at all today.

Making a smartphone and extending the browser

Mozilla is building a smartphone using web technologies, and whenever something is impossible they extend the browser - making it possible.
Might be hard to imagine this web phone competing with native mobile apps, but think about the desktop. Before I only read newspapers and checked the weather on the web. Today I read all my email, write all my documents and even watch TV in the browser. Today the browser is the best development environment on desktop for nearly every type of application and why should mobile be any different?

Packaging webapps

  1. Zip the resources (including the manifest.webapp)
  2. Create an external mini-manifest called package.webapp
  3. Install the app with navigator.mozApps.installPackage(url-to-mini-manifest)
